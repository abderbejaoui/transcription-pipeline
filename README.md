# Medical Voice Corrector — Demo

A self-hosted demo that listens to a doctor speak, transcribes with Whisper,
and corrects misheard medical terms by combining:

- **Whisper** (faster-whisper) for the raw transcript with word-level timestamps
- **wav2vec2-base** for per-word audio embeddings ("voice fingerprints")
- **An LLM** (your Ollama endpoint) for two structured calls:
  - **DETECT** — find suspicious medical-term spans
  - **DECIDE** — pick the right candidate from voice + description matches,
    or refuse with `NO_CHANGE`

When you correct a wrong word in the UI and click Save, the audio fingerprint
of that word is stored. Next time the same sound appears, it's recognised
even if Whisper writes the text completely differently.

## Requirements

- Python 3.11
- `ffmpeg` on `$PATH` (used to decode browser-recorded `webm/opus` to 16 kHz)
- Reachable Ollama endpoint with the model in `OLLAMA_MODEL` (defaults below)

## Setup

```bash
git clone <this repo>
cd test_sound_pipeline
```

## Configuration

Environment variables (all optional, sensible defaults):

| Var | Default | What |
|---|---|---|
| `WHISPER_MODEL_SIZE` | `base` | tiny / base / small / medium / large-v3 |
| `WHISPER_DEVICE` | `cpu` | `cuda` on Linux+GPU |
| `WHISPER_COMPUTE_TYPE` | `int8` | `float16` on GPU |
| `WHISPER_LANGUAGE` | `en` | force language; UI dropdown overrides per-request |
| `VOICE_ENCODER_MODEL` | `facebook/wav2vec2-base` | any wav2vec2 checkpoint |
| `OLLAMA_URL` | `http://100.68.87.28:11434/api/chat` | your Ollama endpoint |
| `OLLAMA_MODEL` | `hf.co/bartowski/calme-3.2-instruct-78b-GGUF:IQ4_XS` | model tag |
| `USE_LLM` | `1` | set to `0` to skip both LLM calls (text-only fallback) |

Confirm Ollama is reachable before you start:

```bash
curl -sS $OLLAMA_URL ; echo
curl -sS "${OLLAMA_URL%/chat}/tags" | head -1
```

## Run

```bash
source .venv/bin/activate
uvicorn app.main:app --host 127.0.0.1 --port 8000
```

Open <http://localhost:8000>.

First boot pre-warms two models in the background:

- `faster-whisper '<size>' ready.`
- `[voice_match] loading facebook/wav2vec2-base on <device>`

Initial wav2vec2 weights download is one-off (~95 MB).

## Seed the description cache (optional, recommended)

Each medical term in `data/medical_lexicon.jsonl` benefits from a one-line
clinical description. The DECIDE step uses these to disambiguate
candidates by meaning. They are generated by the LLM and cached in
`data/descriptions.jsonl`.

```bash
source .venv/bin/activate
SEED_PARALLEL=4 python -m scripts.seed_voice_db --skip-voice
```

Roughly 4 descriptions / minute through Calme-78B → ~50 minutes for 200
terms. Idempotent: re-running picks up where it left off.

## How to test

1. Open <http://localhost:8000> and click 🎙️ **Record**.
2. Read a sentence with a medical term, e.g.
   *"The patient with asthma uses salbutamol."*
3. Whisper will probably misspell the drug. The pipeline shows a live
   trace panel: `asr.done` → `detect.request` / `detect.response` →
   `retrieve.span` (voice DB hits) → `decide.request` / `decide.response`
   → `final`.
4. The first time, the voice DB is empty for your voice, so DECIDE returns
   `NO_CHANGE`. Edit the corrected box to the right form and click 💾
   **Save edits**. The audio fingerprint of that word is stored under the
   canonical term.
5. Record the same sentence again. The trace now shows
   `voice_match: salbutamol@0.94` in the `retrieve.span` event and
   DECIDE picks it.

## Resetting state

```bash
# wipe everything the pipeline has learned (voice fingerprints + sessions)
rm -f data/voice_index.npz data/voice_index.jsonl
rm -rf data/sessions
# also wipe LLM-generated descriptions if you want to re-seed
rm -f data/descriptions.jsonl
```

Full DGX run:

```bash
.venv/bin/python scripts/prepare_dgx_full_asr_dataset.py \
  --work-dir data/dgx_full \
  --confirm-full-download
```

Optional neighbor-Gulf augmentation with WorldSpeech Kuwait/Bahrain:

```bash
.venv/bin/python scripts/prepare_dgx_full_asr_dataset.py \
  --work-dir data/dgx_full \
  --confirm-full-download \
  --include-neighbor-gulf
```

To include the optional Mansour Emirati cartoon corpus after confirming rights:

```bash
.venv/bin/python scripts/prepare_dgx_full_asr_dataset.py \
  --work-dir data/dgx_full \
  --confirm-full-download \
  --include-mansour
```

The Mansour path parses Gulf Arabic Resources PDFs, extracts timestamps and
dialogue, downloads linked YouTube audio only when requested by the DGX flag,
and rejects timestamp spans outside the same 1-30 second ASR limit.

The preprocessor outputs 16 kHz mono PCM16 WAV clips with normalized
Arabic-English transcripts and grouped train/validation/test manifests for the
full DGX pipeline. The DGX script also writes
`data/dgx_full/download_hours_summary.json` immediately after downloading so
contributors can verify total hours per dataset before preprocessing.

## Known limitations

- **First occurrence of a new term is unfixable.** Voice DB has no fingerprint
  for it yet. DECIDE returns `NO_CHANGE`. User fixes once → next time it's
  recognised.
- **wav2vec2-base has speaker bias.** A different speaker saying the same word
  may not match the saved fingerprint (cosine 0.7–0.8 instead of 0.9+).
  We require ≥ 0.90 for the voice-first scan to avoid false positives.
- **Two LLM calls per transcript.** With the 78B model, expect 2–6 minutes for
  a 30-minute conversation. A smaller model (e.g. 14B) reduces this 5×.
- **Long audio is not chunked.** A single DETECT call covers the whole
  transcript; very long inputs may exceed the LLM's effective context. If
  you need >5 minutes per call, add chunking in `_run_transcribe_pipeline`.
